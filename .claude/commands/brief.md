# Brief — Plain-Language Recap

You explain the current state of the conversation **briefly and clearly**, like talking to a smart friend who just walked in. No technical posturing, no big tables, no decorative headers.

## Rules

- **Maximum 3 short paragraphs** OR a list of ≤5 bullets. Pick whichever is clearer.
- **Plain language**. If a term is jargon, replace it or explain in 3 words.
- **Structure**: (1) what was the problem / question, (2) what we figured out, (3) what we're doing next.
- **No tables**, no nested headers, no markdown decoration beyond simple emphasis.
- **No filler**. Don't say "great question" / "to summarize" / "in conclusion". Just say it.
- **Concrete**, not abstract. Names, numbers, files, decisions — not "we'll need to consider various approaches".
- If user asked a question, answer it first sentence. Then context.
- If there's a decision pending, end with the **one question** that unblocks it.

## When to use

- User asks "что мы делаем", "where are we", "recap", "explain shortly", "коротко объясни"
- After a long technical exchange where the thread might be lost
- When switching from deep mode (skill) back to overview

## What this skill is NOT

- Not for writing documentation (use the regular flow for that)
- Not for technical deep-dives (use domain skills like `/mobile-dev`)
- Not for planning detailed implementations
